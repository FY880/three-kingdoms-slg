## 2. 核心设计支柱

### 2.1 武将战法系统

#### 2.1.1 四类型框架（设计目标 = 率土/谋定同构）

| 类型 | 触发时机 | 结算入口 | 当前实现状态 |
|---|---|---|---|
| **指挥** | 战斗开场一次性结算 | `skills.js → commandSetup`（`sk.type==='指挥'`） | ✅ 已实现（如「奸雄」） |
| **被动** | 开场常驻标记 + 每回合 tick | `commandSetup`（`sk.type==='被动'`）+ `tick` | ✅ 已实现（如「仁德」「空城」） |
| **主动** | 每回合按 `fireRate` 概率触发 | `simulateCombat` 循环内 `cast` | ✅ 已实现（9 个主动战法） |
| **追击** | 普攻后触发（设计目标） | — | ⚠️ **已知缺口**：引擎 `skill.type` 字段与 Demo UI 标签（`追击:'追'`）已预留，但**配表 0 个追击战法**，`simulateCombat` 循环也只 `if (sk.type!=='主动') continue`，**普攻后追击 hook 未建** |

#### 2.1.2 发动率公式（`skills.js:22` `fireRate`）

```
发动率(%) = clamp( round( skill.rate + (stat − 80) × 0.22 + bondRate × 100 ), 25, 92 )
```

- `stat`：谋略战法取 `g.intellect`，兵刃战法取 `g.force`（按 `sk.school` 分支）。
- `bondRate`：来自 `applyTeamBonus` 的「阵营羁绊率 + 武将缘分率」之和（如全魏阵 +5%、桃园结义 +8%）。
- 钳制 `[25, 92]`：低属性不会低于 25%，高属性/高羁绊不会超过 92%。

**⚠️ 已知细节（非 bug，设计需确认）**：`fireRate` 取的是配表**原始属性** `caster.g.force / caster.g.intellect`，与 `makeUnit` 的等级成长 `grow` **相互独立**——升级放大攻防与伤害，但**不直接提升发动率**。若希望「升级同步提升发动率」，需改为读 `caster.force / caster.intellect`（已成长的数值）。

#### 2.1.3 控制 / 持续伤 / 增减益 / 反弹 / 免疫

| 机制 | 字段 / 入口 | 行为 | 状态 |
|---|---|---|---|
| 震慑（stun） | `ctrl:'stun'` | 本回合跳过全部行动（`simulateCombat` 特判 `continue`） | ✅ |
| 沉默（silence） | `ctrl:'silence'` | 封锁主动战法（`if (u.ctrl!=='silence')` 才开火） | ✅ |
| 混乱（confuse） | `ctrl:'confuse'` | 普攻误击友军（`Math.random` 选友方） | ✅ |
| 缴械（disarm） | `u.disarmed` | 封锁普攻（`if (!u.disarmed)` 才普攻） | ✅ |
| 震慑（shock） | `ctrl:'shock'`（威震华夏/鹰视） | ⚠️ **已知缺口**：`applyControl` 会写入 `u.ctrl='shock'`，但 `simulateCombat` 循环**未特判 `shock`**（仅特判 stun/silence/confuse/disarm），当前 `shock` 在行动限制上≈空操作，需补分支 | ⚠️ |
| 持续伤（dot） | `dotPct/dotSchool/duration` | 火攻/火烧连营施加灼烧，`tick` 每回合结算并递减 | ✅ |
| 增益/减益 | `buffAtkPct/buffDefPct/debuffAtkPct/debuffDefPct` | 乘入 `atkMul/defMul` | ✅ |
| 偷取（steal） | `stealPct`（鹰视） | 偷取敌方攻击并自加 | ✅ |
| 反弹（reflect） | `reflect`（制衡） | 置 `reflectNext` 标记（反弹就绪） | ✅ 标记已置，结算链路待战斗循环接反伤 |
| 免疫控制 | `immuneCtrl`（龙胆/空城） | 首次受控时消耗 `immuneCtrlUsed` 免疫一次 | ✅ 一次性免疫正确 |

#### 2.1.4 指挥与被动的开场结算（`skills.js:43` `commandSetup`）

战斗开始前，`applyTeamBonus`（羁绊/缘分）先跑，`commandSetup` 再对每单位结算其**指挥/被动**战法：

- **指挥**：对全军乘算 `atkMul *= (1+buffAtkPct)`、`defMul *= (1+buffDefPct)`、加士气（`buffMorale×20`，钳制 20~100）。
- **被动**：`immuneCtrl` 置位、`defMul` 常驻加成、`foeHitDown` 叠加（降敌命中）、`regenPct`（伤兵恢复）、`regenMorale`（士气恢复）。

**⚠️ 描述与实现漂移（待对齐）**：

- 「奸雄」描述为「**每回合**提升全军攻击」，实现是**开场一次性**指挥加成（非每回合刷新）。
- 「空城」描述为「**残血时**大幅提升防御」，实现是**开场无条件常驻** `+25% defMul` + 敌方 `-10% 命中`（无血量阈值分支）。
- 两者建议二选一对齐：要么改描述，要么实现「每回合/残血阈值」逻辑。

### 2.2 兵种克制

#### 2.2.1 克制链（`troops.csv` 的 `beat` 字段 + `combat.js:58` `troopRPS`）

| 兵种 | beat（克） | 被克 | 克制系数 | 说明 |
|---|---|---|---|---|
| 骑（骑兵） | 步 | 弓 | **攻击骑克步 → 1.15**；步克骑 → 0.88 | 平原/荒漠逞威 |
| 步（步兵） | 弓 | 骑 | **步克弓 → 1.15**；弓克步 → 0.88 | 稳守正面 |
| 弓（弓兵） | 骑 | 步 | **弓克骑 → 1.15**；骑克弓 → 0.88 | 林地/丘陵射程佳 |
| 车（车兵） | （空） | — | 不参与三角 → 双方均 1.0 | 野战平庸，**攻城专精** |
| 械（器械） | （空） | — | 不参与三角 → 双方均 1.0 | 野战怯弱，**摧城无双** |

**三角链**：骑 → 步 → 弓 → 骑（1.15 进攻方 / 0.88 被克方）。车、械定位为攻城，不进三角。

#### 2.2.2 地形适性（`fieldCol`）+ 攻城加成（`siegeBonus`）

- **地形适性**：`troops.csv` 的 `fieldCol`（`cav/inf/arch/eng`）映射到 `TERRAIN[地形][col]` 列（`data.js` 地形表），乘入普攻 `troopAtk`（`combat.js` 普攻链 `env.atk × troopAtk`）。例如骑兵在平原 `cav=1.15`、入山地 `cav=0.7` 战力骤降——「在哪打、用什么兵」成为真决策。
- **攻城加成**：`siegeBonus` 仅在 `FORTIFIED`（城 `city` / 隘 `defile` / 桥 `bridge`）上生效（`combat.js` 普攻链 `siege`），车 **1.3**、械 **1.6**，其余地形/兵种为 1.0。对应《率土》《谋定》的「器械克城防」。

> 普攻伤害乘子链（`combat.js:194`）：`atkMul = formation.atk × suitCoef(suit) × env.atk × troopAtk × rps × siege × styleCoef × (morale/80) × choke(攻方) × ambush(首回合一击) × sudden(猝死) × u.atkMul × (1 − foeHitDown)`。所有系数均数据驱动，可由 CSV 调。

### 2.6 武将等级成长（`combat.js:88` `makeUnit`）

```
grow = 1 + (lv − 1) × 0.06
force/intellect/leadership/speed = round( 配表原始值 × grow )
```

- 作用于 **力 / 智 / 统 / 速** 四类（对应 `force/intellect/leadership/speed`）。
- 当前配表武将多为 `lv=5`（孙权/黄忠 `lv=4`）→ `grow(5) = 1.24`。例：关羽 `force 97 → 120`、吕布 `force 100 → 124`。
- 与 `fireRate` 的关系见 2.1.2 的 ⚠️ 已知细节（发动率不吃 `grow`，只吃原始属性）。

## 3. 数据驱动架构说明

### 3.1 三层解耦（逻辑 / 数据 / 表现）

```
┌─────────────────────────────────────────────┐
│ 表现层（可替换）                               │
│  index.html（Canvas Demo）/ Godot / Unity / Cocos│
├─────────────────────────────────────────────┤
│ 聚合入口：src/rules.js → window.RULES          │
│  ├─ combat（环境/兵书/战斗/阵法）  simulateCombat/makeUnit/chooseFormation│
│  ├─ skills（战法引擎）           fireRate/cast/tick/commandSetup        │
│  ├─ economy（资源）              starting/perTurn/recruitCost/...        │
│  ├─ diplomacy / scenario（关系/剧本）                                   │
│  └─ data（配表 + 热加载 + 可变 state）                                 │
├─────────────────────────────────────────────┤
│ 数据层：data/*.csv（9 张，与代码零耦合）       │
└─────────────────────────────────────────────┘
```

### 3.2 纯逻辑层（零 DOM）

- `src/*.js` 全部为 **UMD**：浏览器 `<script>` 挂全局（`RULES_*`）、Node `require` 双兼容。
- 逻辑层**不触碰任何 DOM**，`simulateCombat / makeUnit / applyTeamBonus / fireRate / troopRPS / ECON.*` 均为纯函数，输入参数、返回结果，可被任意引擎直接调用。

### 3.3 配表热加载（可变 state + getter）

- `data.js` 用**可变 `state` 对象**承载所有表；`reload()` 只重赋值 `state.*` 属性（不换对象引用），其余模块在「调用时」读 `state.*` → **热加载即时全局生效**。
- `rules.js` 用 `Object.defineProperties` 把 `RULES.TERRAIN / SKILLS / ...` 暴露为 **getter**，因此 `reload()` 后 `RULES.SKILLS` 自动读到新表。
- 两种注入入口：`loadConfigFromServer(base)`（浏览器 fetch CSV）、`loadCSVFromDir(dir)`（Node 读 CSV，供平衡脚本）。

### 3.4 单文件离线 Demo（`tools/build-standalone.js`）

- 把 `src/` 8 个模块内联 + `data/` 9 张 CSV 嵌成字符串（`window.__CSV__`）。
- 启动时 `parseCSV + reload` 注入内存表，并 monkeypatch `R.loadConfigFromServer` 走内嵌数据 → **「配表」按钮离线也能热重载**。
- 产物 `dist/index.standalone.html`：**单文件、零 fetch、零服务器**，手机/电脑浏览器直接打开（`file://` 亦可）。

### 3.5 引擎移植接口（Godot / Unity / Cocos）

外部引擎只需 4 步，逻辑层整体平移：

1. `require/import` 聚合入口拿到 `RULES`（`rules.js`）；
2. 调用纯函数（`RULES.simulateCombat / makeUnit / applyTeamBonus / ECON.* …`）；
3. 用 `loadCSVFromDir / loadConfigFromServer` 注入 `data/*.csv`（配表与代码分离，调平衡不动引擎）；
4. 渲染与输入由引擎自管，**逻辑层零 DOM 依赖**。

> 这与旧版总纲 §17.6 的移植设计一致，本轮深度策略支柱完全复用该架构，无需改造。
