# 三国 SLG 单机版 · 游戏设计文档（GDD v1.0）

> 类型对标：率土之滨 / 三国谋定天下 的**单机化改编**
> 形态：**单机 / 离线可玩**（无服务器，AI 替代真人玩家）
> 文档定位：给**个人独立开发者**的落地蓝图——可先砍成 MVP 逐步迭代
> 文档状态：v1.0 框架版，文中所有数值均为「示例初值」，需后续调平衡

---

## 1. 项目概述

### 1.1 一句话定位
> 在三国乱世的一张大地图上，你扮演一方诸侯，招募武将、搭配战法、连地扩土、与 AI 势力合纵连横，最终完成「霸业目标」统一天下。

### 1.2 单机化核心改编思路

| 原版（多人 MMO） | 单机版替代方案 | 设计取舍 |
|---|---|---|
| 数千真人玩家抢地盘 | **8~12 个 AI 诸侯势力**，各有性格与扩张 AI | 保留「抢地盘」的领地扩张爽感 |
| 真人同盟协作打城 | **外交系统**：结盟 / 停战 / 附庸 / 背盟 | 保留了谋略与背叛的戏剧性 |
| 真人赛季节奏 | **剧本模式**（黄巾、讨董、赤壁、鼎立）+ 沙盒征服 | 用历史节点提供目标感 |
| 实时在线对抗 | **时间推进制**（每「时辰」结算一次 AI 行为） | 可暂停、可存档，适合单机 |
| 付费抽卡 | **免费获取 + 史实剧情解锁武将** | 单人无需付费点，靠收集驱动 |

### 1.3 核心体验关键词
**扩张感**（连地铺路把地盘连成片）→ **养成感**（武将升级、战法学满）→ **谋略感**（打谁、和谁结盟、何时背盟）→ **翻盘感**（劣势下靠阵容克制逆风翻盘）。

## 2. 核心玩法循环（Core Loop）

```
┌─────────────────────────────────────────────────────────┐
│  每日/每时辰循环（单机时间推进制）                          │
│                                                           │
│  看地图(敌我态势) → 决策(打哪块地/调哪支部队)              │
│        ↓                                                  │
│  出征/驻守/调动  ──→  战斗结算(战法演算)                   │
│        ↓                                                  │
│  占领土地 → 资源产出↑ → 升级建筑/练兵/学战法               │
│        ↓                                                  │
│  武将变强 → 打更高级土地/AI 城池 → 势力值↑                 │
│        ↓                                                  │
│  触发外交事件 ←→ 推进剧本目标 → 达成霸业/失败              │
└─────────────────────────────────────────────────────────┘
```

**外层循环（剧本级）**：开荒期（抢低级地）→ 发展期（占资源州、打关口）→ 决战期（打州府/敌主城）→ 结算（割据 / 征服）。

## 3. 系统模块详细设计

### 3.1 大地图系统
> 详细尺度、迷雾、自由扩张与「辽阔感」设计见 **第 11 节**；地形/天气/水攻/天时地利人和见 **第 14 节**。

- **格子地图**：约 60×60 ~ 100×100 格（MVP），终版目标 **200×200 无缝大图**，按「州」划分（冀州、徐州、荆州…），州与州之间有**关口**与**河流**天然屏障。
- **土地分级**：1~8 级（示例）。等级越高产出越高、守军越强。
- **资源带**：地图散落「木材/铁矿/石料/粮草」专属高产地块，以及**资源州**（高级地产区，必争）。
- **连地机制（关键）**：只能对**与己方领地相邻**的地块出征，不相邻需先「铺路」（占中间地块）或建**要塞**做跳板。这是率土类游戏领土战的灵魂，必须保留。

### 3.2 主城 / 建筑系统
- 主城 1 座（可后期建 1~2 座**分城**扩大产能）。
- 核心建筑（每类 1~25 级，示例）：

| 建筑 | 作用 | 升级收益 |
|---|---|---|
| 大本营 | 解锁其他建筑上限、统帅值 | 兵力上限↑ |
| 兵营 | 训练/补兵 | 单武将带兵上限↑ |
| 仓库 | 资源存储上限 | 防被劫掠清空 |
| 农场/伐木场/炼铁厂/采石场 | 对应资源产出 | 每小时产出↑ |
| 校场 | 部队出征队列 | 同时出征队伍数↑ |
|  Market(集市) | 资源按比例兑换 | 应急调配 |

### 3.3 资源系统
| 资源 | 用途 | 主要来源 |
|---|---|---|
| 粮草 | 养兵、行军消耗 | 农场、占领粮地产地 |
| 木材 | 建筑升级 | 伐木场 |
| 铁矿 | 建筑/装备 | 炼铁厂 |
| 石料 | 高级建筑 | 采石场 |
| 铜币 | 招募武将、学战法 | 税收、占领城池 |

> 单机不做「玉璧抽卡」，武将靠**占领名城 + 史实剧情 + 寻访**获取，避免付费设计。

### 3.4 武将系统
- **稀有度**：1~5 星（示例），影响基础属性与自带战法强度。
- **属性**：武力、智力、统率、速度、政治（内政用）、魅力（寻访/外交用）。
- **兵种适性**：S/A/B/C，决定该武将带某兵种（步/骑/弓/车/器械）时的属性加成。
- **获取途径**：占领历史名城（如占洛阳得某将）、剧情解锁、寻访（消耗铜币+时间）。
- **成长**：升级、进阶（升星，吃同名/同阵营）、加属性点。

### 3.5 战法系统（本作核心爽点，重点做）
- **自带战法**：每个武将天生携带 1 个，不可替换（如关羽「威震华夏」）。
- **学习战法**：拆其他武将 / 战法库习得，装配到空槽（示例每将 1 个空槽，5 星将 2 个）。
- **战法分类**：

| 类型 | 触发 | 示例效果 |
|---|---|---|
| 指挥 | 战斗开始固定生效 | 全军属性↑、先攻 |
| 主动 | 每回合概率触发 | 伤害、控制（混乱/缴械） |
| 被动 | 常驻 | 暴击↑、减伤 |
| 突击 | 普攻后触发 | 追加伤害 |

- **战法等级** 1~10，升级提升数值/概率。克制链：控制类克制爆发类，减伤类克制持续输出类。

### 3.6 部队 / 兵力系统
- 一队 = **1 主将 + 2 副将**（共 3 武将），各带一种兵种。
- **兵力**：战斗单位，受伤变「伤兵」，需回城在时间/资源下恢复（类似率土的伤兵机制，制造节奏感）。
- **统帅值**：限制上场武将总星级，逼你做取舍（不能全上 5 星）。

### 3.7 战斗系统（单机简化演算模型）
> 武将克制、阵法布阵、埋伏/突袭/空城计、将令指挥、兵书机制见 **第 12 节**。

采用**回合制战法演算**（非真人即时，省实现成本），单场示例：

```
每回合：
  1. 按速度决定行动顺序
  2. 触发指挥/被动（常驻加减成已计入）
  3. 普攻：伤害 = 攻击 × 兵种克制系数 × (1 + 战法加成) − 防御减免
  4. 触发主动/突击战法（按概率）
  5. 兵力归零则该武将「败退」
胜：一方三将全败退；败方损失兵力转伤兵
```

- **兵种克制**：骑克步、步克弓、弓克骑（三角），器械克城防、被其他克。
- **地块加成**：守军在自家高级地上享防御加成，鼓励「先铺路再打」。

### 3.8 行军 / 占领系统
| 指令 | 说明 |
|---|---|
| 出征 | 攻击相邻地块守军，胜则占领 |
| 驻守 | 部队停留某地，提供防御 |
| 调动 | 部队在己方领地内快速移动 |
| 要塞 | 消耗资源建临时据点，作为铺路跳板、可屯兵 |
| 集结 | 多支部队围攻同一 AI 城池 |

行军有**时间**（距离 × 兵种速度），可计时的「行军中」状态，符合单机时间推进。

### 3.9 AI 诸侯系统（单机成败关键）
- 地图布置 **8~12 个 AI 势力**，各占一州为起点，君主有性格标签：

| 性格 | 行为倾向 |
|---|---|
| 侵略型（如董卓） | 优先扩张、爱背盟 |
| 稳健型（如刘表） | 守成、少主动出击 |
| 外交型（如刘备） | 倾向结盟、联弱抗强 |
| 霸权型（如曹操） | 滚雪球、中期集火最强者 |

- **AI 行为用状态机 + 简单评分**：评估「附近可打地块收益」「威胁度」「盟友关系」，每时辰决策一次。先做「会铺路、会打城、会结盟」的够用 AI，再迭代难度。

### 3.10 外交 / 同盟系统（单机下的「谋略感」来源）
- 与 AI 势力可：**结盟 / 停战 / 附庸（你或它）/ 宣战 / 背盟（有信誉惩罚）**。
- 信誉值影响其他势力对你的态度（背盟多了人人喊打）。
- 可**借道**（同盟间部队可通过领地）、**联合作战**打第三方。
- 剧本事件会强制改变外交（如「董卓乱政」各路自动讨董）。

> **已落地（Demo / rules.js）**：关系模型以纯逻辑实现 `createDiplomacy / getRelation / setRelation / isAlly / isRival / diploScoreAdj`，并接入 AI 评分 `scoreTarget`——**同盟=互不侵犯（AI 绝不攻你、你也不可攻盟友）**；**敌对=你与盟友优先夹击该诸侯（即「拉一个 AI 打另一个」）**。玩家在「外交」面板切换 同盟/中立/敌对，背盟扣信誉、结盟加信誉；信誉低时更易被诸侯进攻。AI 之间初始按概率随机结盟，使局势自然分化。

### 3.11 剧本 / 赛季系统
> 剧本即「开局配置包」：诸侯阵容、玩家出生点、中立强度倍率、胜利条件、剧本事件，全部作为**数据**放在纯逻辑层（`rules.js` 的 `SCENARIOS`），Demo 侧仅做状态装配与渲染。新增剧本 = 加一条数据，不改逻辑。

| 剧本 id | 名称 | 初始态势 | 胜利目标 | 特色 |
|---|---|---|---|---|
| `melee` | 群雄逐鹿 | 曹/刘/孙/董四军并起，玩家出生西北 | 占领 **60** 州郡（割据→霸业） | 纯实力沙盒，最自由 |
| `yellowturban` | 黄巾之乱 | 黄巾势大，中立守军 ×1.8，玩家居中 | 占领 **50** 州郡 | 逆风开局、以少打多 |
| `dongzhuo` | 讨董卓 | 董卓独占强兵，玩家与诸侯环绕 | **击灭董卓势力** | 第 4 时辰触发 `rivalAll`，各路自发讨董，你亦卷入 |

- **胜利结算分档**：占领类（`type:'land'`，达到 `n` 州郡即「霸业有成」）；击灭类（`type:'defeatLord'`，指定诸侯领地清零即「讨灭」）；玩家领地归零判负。
- **剧本事件（脚本化时序）**：`events:[{at:时辰, type, ...}]`，由 `eventsDueAt(scn,time)` 在每推进时辰时取到期事件，防重复触发。例：讨董的 `rivalAll` 在第 4 时辰把全体 AI 与玩家都设为董卓的敌对（共同讨伐），并写入战报。后续可扩展 `spawnArmy`（黄巾潮）、`buffLord`、`openGate` 等事件类型，实现赤壁东风、官渡奇袭等名场面。

### 3.12 地图视觉表现层（Demo 适配，低成本高辨识）
> 与「宏大地图」的玩法自由度配套，但属于**表现层**，与纯逻辑层解耦。Demo 用纯 Canvas 绘制，零美术资源、零版权风险（文字字形 + 几何描边），移动端 60fps 友好。

- **地形字形**：在格子中央绘制地形汉字（山/林/水/城/泽/洼/丘/峡/沙/雪/桥），缩放低于阈值自动隐藏，避免糊成一片。
- **势力边界**：仅在与邻格归属不同处描边（同归属不画），用不同颜色勾勒出各方领地的清晰轮廓，一眼看清局势。
- **天气表现**：全屏色调 + 轻量粒子（雨丝/雪点/雾团/风线/旱纹）；晴（`sun`）无粒子、不开启 RAF 循环以省电。粒子由 `requestAnimationFrame` 在天气非晴时驱动，约 70 个粒子，性能可控。
- **迷雾与侦察**：未侦察格以深色遮蔽；玩家领地 3 格半径内的格子视为已侦察（`scouted`）。
- **可攻高亮**：与玩家领地相邻的中立/敌方格以黄色描边提示「可出征」（连地规则）。

## 4. 数值框架（示例初值，需调平）

- **资源产出**：n 级地每小时产基础值 ×(1+0.5×n)；主城建筑每级提升约 15%。
- **带兵上限**：大本营 Lv × 200（示例，满级约 5000/将）。
- **战斗时长**：单场 **10 回合**封顶，第 **7 回合**起进入「猝死阶段」，伤害按 `1+(回合-7)×0.3` 递增，保证战斗必收束、节奏明快（详见 §17.1）。
- **伤兵恢复**：回主城后，伤兵以「每小时恢复 10% + 消耗粮草」复原。
- **势力值** = 占领地块数 × 等级权重 + 城池数 × 大权重，用于排名与胜负判定。
- **平衡原则**：前期节奏快（开荒爽），中期卡资源（逼你打资源州），后期拼阵容克制（防数值碾压）。

## 5. 单机化关键设计决策

1. **时间推进制**：用「时辰/天数」推进，AI 每 tick 决策，玩家可暂停思考——这是单机相对 MMO 的体验优势（不焦虑、可随时存档）。
2. **AI 难度档**：简单/普通/困难，调节 AI 扩张速度与战斗加成，单人易调试。
3. **离线收益**：关闭「离线挂机收菜」或仅给极低被动产出，避免数值通胀；单机乐趣在决策而非收菜。
4. **防作弊无意义**：单机无需反外挂，但保留「新游戏+」或调试开关方便开发自测。
5. **存档**：本地单文件存档（JSON/SQLite），支持多剧本多周目。

## 6. MVP（给个人开发者的第一版最小可玩）

> 目标：**2~4 周内能跑通一个「能玩」的纵向切片**，验证核心乐趣，再扩内容。

| 优先级 | 必做 | 可后做 |
|---|---|---|
| P0 | 格子地图 + 连地出征 + 占领 | 资源州/关口美术 |
| P0 | 主城 3~4 建筑 + 资源产出 | 分城 |
| P0 | 5~10 个武将 + 自带战法 + 战斗演算 | 战法学拆、装备 |
| P0 | 1 个 AI 势力会铺路/打你 | 多性格 AI、外交 |
| P1 | 1 个剧本（讨董）+ 胜利判定 | 多剧本、剧情 |
| P1 | 本地存档 | 云存档（单机无需） |

**MVP 不做**：抽卡付费、多人联机、3D、复杂剧情演出。

## 7. 技术实现建议（单机 / 个人）

| 层 | 建议 | 说明 |
|---|---|---|
| 引擎 | **Godot 4**（2D，免费开源）或 Unity | 格子 SLG 用 2D 足够，Godot 对单人最友好 |
| 语言 | GDScript / C# | 逻辑为主，性能压力小 |
| 地图 | TileMap + 二维数组存地块状态 | 100×100 也就 1 万格，内存无压力 |
| 存档 | 本地 JSON / SQLite | 单文件、易调试 |
| 战斗 | 纯逻辑模块（与渲染解耦） | 方便单元测试与平衡 |
| 美术 | 先占位色块 / 免费素材，后替换 | 单人别卡在美术上 |
| 数据 | CSV/JSON 配表（武将、战法、地块） | 数值与代码分离，好调 |

> 关键：**战斗演算、AI 决策、数值全部做成可配置的纯逻辑层**，UI 只是壳。这样你改数值、加武将都不用动代码。

## 9. 风险与验证

- **最大风险**：AI 不够「像人」，抢地盘失去张力 → 先用状态机做「会扩张会结盟」的够用 AI，靠性格标签制造差异。
- **次风险**：数值崩（前期太慢 / 后期碾压）→ 所有数值走配表，M1 就建平衡测试。
- **验证方式**：自己每天玩 30 分钟 MVP，记录「哪里无聊 / 哪里卡」；先让 3~5 个朋友试玩讨董剧本。

## 10. 合规与侵权规避（必读，上线前必查）

> 说明：以下为面向个人开发者的**实务风险边界**，不是法律意见；真要商业化发行，建议花一笔小钱咨询知识产权律师。

### 10.1 先分清「能自由用」与「不能碰」

| 类别 | 风险等级 | 结论与做法 |
|---|---|---|
| **三国历史事实、年代、战役**（如赤壁、官渡） | ✅ 安全 | 历史事实不受版权保护，随便用 |
| **历史人名**（曹操、刘备、诸葛亮、关羽…） | ✅ 安全 | 历史真实人物姓名不受版权保护，可用 |
| **游戏玩法/规则机制**（连地出征、战法回合演算、伤兵恢复） | ✅ 安全 | 「思想/规则」不保护，模仿机制合法；这是 SLG 品类的公共养料 |
| **具体美术资产**（立绘、3D 模型、地图贴图、UI 图标） | ⛔ 高危 | **绝对不能**从率土/谋定天下或任何游戏里扒图；用自己的或合法授权素材 |
| **具体商标名**（「率土之滨」「三国谋定天下」及 Logo） | ⛔ 高危 | 不能用作你游戏的**名称、副标题、宣传语、商店关键词**；避免「率土」「谋定」等近似词 |
| **具体文字表达**（某游戏里某武将的**专属战法原文描述、台词、角色生平文案**） | ⚠️ 中高 | 「思想」不保护，但**具体措辞**受保护；自己重写战法名与描述，别照抄 |
| **音乐 / 音效** | ⚠️ 中高 | 别用原声带；用 royalty-free 或自制的 |
| **角色独特形象设计**（某游戏独创的兵器/服饰/配色组合） | ⚠️ 中 | 历史风装扮 OK，但别像素级复刻某游戏的「独家人设」 |

### 10.2 给你的 6 条落地红线

1. **换皮不换骨，但名字/美术/文案全原创**：机制可以借鉴，但武将战法名、描述、立绘、剧情文案必须自己写。例如战法叫「威震华夏」若与某作完全一致且有独创表述，就改个名重写成你自己的版本。
2. **游戏名与商店关键词避开对标品**：别叫「XX率土」「XX谋定天下」；标题走原创（如《烽火三国》《群雄逐鹿·单机版》）。
3. **美术来源三选一**：① 自己画；② AI 生成（注意平台对 AI 图的政策）；③ 明确可商用的免费素材（CC0 / 购买授权）。严禁「截图→描图」对标游戏。
4. **引擎许可看清**：Godot（MIT，宽松）、Cocos（宽松）可放心；Unity 注意其**运行时收费（Runtime Fee）**条款与收入阈值，个人低风险但上线前核对当前版本协议。
5. **历史人物可塑，但别碰在世人物或现代敏感题材**：三国全是古人，安全。不要借三国影射现实敏感内容。
6. **发行平台政策**：上 Steam / 微信小游戏 / 手机商店前，各平台有版号/资质要求（尤其国内移动端需**游戏版号**），单机不排除内购也要合规；个人出海（Steam）相对宽松但仍有成人内容等规则。

### 10.3 降低风险的「原创化」加分项
- 在三国框架上做**轻度架空**：加入原创势力、原创女将/虚构将领、原创事件，稀释「纯复刻」观感，也利于做差异化卖点。
- 战法/武将**文案建立自己的世界观语言**，形成品牌辨识度（玩家记住的是你的表达，不是历史本身）。
- 保留一份**素材授权清单**（每张图/每段音乐的来源与许可），上线前自查，出事能举证清白。

### 10.4 一句话原则
> **抄机制不抄表达，用人名不抄人设，借历史不借商标。** 做到这三点，个人单机三国 SLG 的侵权风险可控制在很低水平。

## 11. 宏大地图与战略自由度（扩张爽感）

> 设计目标：让玩家从「一块地」开始，靠自由决策把色块一点点连成「半壁江山→整张地图」，并始终对未知的大地图保有探索与掌控欲。

### 11.1 地图尺度与分层
- **超大无缝地图**：建议 200×200 起步（4 万格），可用**程序化生成 + 历史名城锚点**（洛阳、长安、建业等固定位置），兼顾「辽阔」与「有迹可循」。
- **四级结构（缩放下钻）**：
  | 层级 | 内容 | 玩家视角 |
  |---|---|---|
  | 天下图 | 13 州轮廓、势力色块、主城 | 战略全局，「我在哪、敌在哪」 |
  | 州/郡图 | 城池、关口、资源州 | 选扩张方向 |
  | 战区图 | 地块格、部队、要塞 | 排兵布线 |
  | 战斗图 | 双方阵列、地形 | 将令微操 |
- **领土涂色**：己方连片领地实时染色（类《欧陆风云》），占领越多视觉越爽，是核心正反馈。

### 11.2 战争迷雾（自由度的前提）
- 只显示**已侦察**区域：己方领地周围、部队视野、派出细作/侦察兵的范围可见；远处为迷雾。
- 迷雾带来：未知敌情 → 探索欲；伏兵/突袭/空城计有了成立基础（详见第 12 节）。
- 侦察来源：主城/要塞辐射、行军部队视野、侦察骑兵、细作潜入。

### 11.3 自由扩张与战略要地
- **无强制路线**：连地规则保留（只能打相邻地块），但你可以自由选择「先铺哪条线、绕哪座城、抄哪条小道」。
- ** choke 点（要害）**：关口、渡口、栈道、桥——控制它们就能「一夫当关」卡死敌方大军团，逼出迂回/暗度陈仓玩法。
- **资源州 + 产粮大泽**：地图必争之地，天然制造冲突与「打到那里才爽」的目标。
- **纵深与补给线**：占地越远离主城，补给越吃力（粮草随时间衰减），迫使你建**要塞链**做跳板——这正是「布线」的爽点。

### 11.4 给「辽阔感」的辅助设计
- 占领整州有**州府加成**（全州产出↑），给阶段性里程碑；统一天下触发终局演出。
- AI 也用迷雾，你与 AI 互相看不见深处，情报战成为常态。

## 12. 将帅指挥 · 武将克制 · 诡道战法（兵书玩法）

> 设计目标：让玩家像真实将帅一样「运筹帷幄」——排兵、设伏、突袭、用诈，并把《孙子兵法》《三十六计》等兵书变成可操作的机制，而非背景文字。

### 12.1 武将克制（三维克制模型）

不止兵种三角，再叠两层：

**(1) 输出/防御属性克制（核心）**
| 攻击类型 | 由谁打出 | 被谁减伤 | 克制关系 |
|---|---|---|---|
| 兵刃伤害 | 武力 | 统率 | 高武力将 克 低统率脆皮 |
| 谋略伤害 | 智力 | 智力 | 高智谋将 克 低智武力将 |
> 即「菜刀怕谋士，谋士怕统将，统将怕菜刀」的三角，逼你**阵容互补**而非堆一种。

**(2) 将道风格克制（RPS）**
| 风格 | 特点 | 克 | 被克 |
|---|---|---|---|
| 鹰扬（进攻） | 爆发高、续航弱 | 持重 | 奇变 |
| 持重（防守） | 减伤高、机动低 | 奇变 | 鹰扬 |
| 奇变（诡道） | 控制/迂回、怕硬刚 | 鹰扬 | 持重 |

**(3) 将领特质**：如「多疑」（侦察↑、怕反间）、「刚烈」（攻↑防↓）、「短虑」（易被伏）、「沉稳」（抗控），让克制更有个性。

### 12.2 排兵布线 · 阵法与布阵
- **阵法（开战前选）**：八阵图、锋矢、鹤翼、方圆、雁行、鱼鳞等，各给明确数值：
  | 阵法 | 效果（示例） |
  |---|---|
  | 锋矢 | 中央突破+攻，侧翼脆 |
  | 鹤翼 | 两翼包抄+侧击伤害 |
  | 方圆 | 防御+反伤，机动- |
  | 八阵 | 全属性小幅+抗控（需高智将领） |
- **战区布阵**：三将可摆「前/中/侧」相对位置——前排先接敌承伤、侧翼享包抄加成、后排谋士安全输出。阵型克制敌方布阵（如鹤翼克锋矢侧翼）。
- **布线（地图层）**：用要塞/驻军连成「补给线」与「防线」，决定援军到达速度，是宏观的「排兵」。

### 12.3 埋伏 · 突袭 · 空城计（诡道三件套）
| 机制 | 触发/规则（示例） | 兵书出处 |
|---|---|---|
| **埋伏** | 山林/峡谷格可置「潜伏」部队（地图上不显形）；敌部队进入且未被其侦察发现→我先手+伤害加成 | 孙子·地形、三十六计·暗度陈仓 |
| **突袭** | 轻骑/高速部队对「侦察度低」目标发动，跳过部分城防、直击后方粮仓/要塞，劫粮；敌侦察高则触发警报减伤 | 孙子·虚实、三十六计·趁火打劫 |
| **空城计** | 主城/要塞可「藏兵」显示虚假低兵力诱敌；敌来攻时伏兵/援军触发反击，并使其「惑」状态（命中↓） | 三十六计·空城计 |
| **疑兵/声东击西** | 派小股部队佯动吸引敌军主力方向，真实主力从另一侧突破 | 三十六计·声东击西 |
| **诈败/走为上** | 部队假败退引敌入伏击圈；残血可「走为上」脱离，保留兵力 | 三十六计·走为上、金蝉脱壳 |
| **反间** | 用间使敌将临时倒戈/内讧，或降低其忠诚 | 三十六计·反间计、用间篇 |

> 这些「诡道」做成**可释放指令/战法**，消耗「诡道点」（由高智谋将、夜战、山林地形提供），让谋略型玩家有专属操作空间。

### 12.4 将帅掌控感（指挥系统）
- **你是统帅，不直接控小兵**：发布「军令」——全军突击 / 坚守 / 迂回包抄 / 且战且退，部队按令自动驾驶。
- **将令微操**：战斗中可消耗「令」下达变阵、集火某将、火攻某格、撤某一翼；体现「临机决断」。
- **军团/方面军**：多支部队编为一个军团，设副将代行指挥，打大战役时分层管理（你管战略，副将管战术）。
- **知己知彼**：细作/侦察提供敌「兵力、主将、阵型」情报面板，呼应孙子「知己知彼，百战不殆」。

### 12.5 兵书玩法系统（把经典拆成机制）

**(A) 《孙子兵法》十三篇 → 系统映射**
| 篇 | 落地为游戏机制 |
|---|---|
| 计篇 | 战前「庙算」面板：双方势力/兵力/地利对比，预测胜率 |
| 作战 | 后勤系统：远征吃粮、补给线、劫粮 |
| 谋攻 | 「不战而屈」：威慑/劝降，使弱敌不战而降 |
| 军形 | 防御建设：城防、要塞等级 |
| 兵势 | 奇正：正兵牵制+奇兵决胜的编队 |
| 虚实 | 避实击虚：打击敌方空虚后方/援军未到时进攻 |
| 军争 | 争险：抢关口、高地、渡口的先手权 |
| 九变 | 应变指令：随机应变战术（弃守/改道） |
| 行军 | 扎营/宿营：地形选择影响恢复与视野 |
| 地形 | 六地形（通/挂/支/隘/险/远）各带战斗修正 |
| 九地 | 深入敌境「死地则战、散地则走」的士气/增益 |
| 火攻 | 火攻指令：烧粮草/林地，地形持续伤害 |
| 用间 | 细作体系：反间、乡间、内间获取情报 |

**(B) 《三十六计》→ 可释放「诡道指令」库**（节选代表）
瞒天过海（伪装调动）、围魏救赵（攻其必救迫使回防）、借刀杀人（诱使第三方打你敌）、以逸待劳（守株待敌疲再击）、声东击西、无中生有（虚张兵力）、暗度陈仓、隔岸观火、笑里藏刀（假盟真袭）、调虎离山、欲擒故纵、擒贼擒王（集火主将）、釜底抽薪（断补给）、浑水摸鱼、金蝉脱壳、关门捉贼、远交近攻、假道伐虢、反间计、苦肉计、空城计、连环计、走为上。

**(C) 其他兵书 → 「兵书研读」科技树（被动/解锁）**
- 《六韬》《三略》：统帅类被动（军团上限↑、指令消耗↓）
- 《尉缭子》：治军（伤兵恢复↑、士气稳）
- 《司马法》：军礼/仁义（占城后民心↑、叛乱↓）
- 《李卫公问对》：阵法精通（阵法效果↑、可混搭）
- 《鬼谷子》《阴符》：纵横/用间（外交成功↑、细作效力↑）

> 兵书以「研读进度」解锁，既做成长线收集，又把古人智慧变成**可玩系统**——这是本作区别于普通 SLG 的最大卖点。

### 12.6 给个人开发者的实现提醒
- 诡道/兵书机制**先做成「数据驱动的指令表」**：每个指令 = 触发条件 + 效果 + 消耗 + 视觉表现，配表扩充，不写死逻辑。
- MVP 先实现「埋伏 + 突袭 + 空城计 + 阵法」4 个最有「将帅感」的，其余兵书指令按研发节奏逐步加。

## 14. 地形 · 天气 · 天时地利人和系统

> 设计目标：古代战争胜负常系于「天时地利人和」。本作把它们做成**可计算的环境变量**，叠加在战斗/行军结算上，让大地图从「平面格子」变成有呼吸、有意外、有史诗感的真实战场。

### 14.1 地利 · 地形系统
每种地形给「移动 / 攻防 / 特殊」三维修正（示例初值，配表可调）：

| 地形 | 移动 | 攻防 | 特殊效果 | 兵种偏好 |
|---|---|---|---|---|
| 平原 | ↑↑ | 中立 | 无遮挡，大会战主场 | **骑兵**攻↑、机动↑ |
| 丘陵 | ↓ | 守方+ | 缓坡易守 | 弓兵射程+ |
| 山地 | ↓↓ | 守方++ / 攻方惩罚 | **天险难攻**，伏兵佳 | 步兵坚防 |
| 森林 | ↓ | 守方+ | **隐蔽**（侦察↓/伏兵+），火攻蔓延强 | 埋伏流 |
| 沼泽 | ↓↓ | 中立 | **行军损兵**（持续伤兵/体力↓），补给难恢复 | 全体忌 |
| 河流/水域 | 阻断 | — | 需渡口/桥；邻低洼可**水攻** | 水军 |
| 低洼/盆地 | ↓ | 守方− | **洪水受害区**，易淹 | 忌驻重兵 |
| 峡谷/隘口 | ↓ | 守方+++ | **一夫当关**，攻方通道窄受罚，伏击极佳 | 扼险流 |
| 城池/关隘 | — | 城防值 | **天险**，攻方攻城惩罚 | 器械克 |
| 荒漠/戈壁 | ↓ | 中立 | **补给消耗↑**（粮草锐减），疫病↑ | 忌久战 |
| 雪原/冰原 | ↓↓ | 中立 | 移速↓、补给耗↑、**干柴利火攻** | 火攻流 |
| 桥梁/栈道 | 单线 | — | **咽喉点**，可断桥/烧栈阻敌 | 控点流 |

> 兵种×地形联动：骑兵在平原如鱼得水、入山地战力骤降；弓兵在森林/丘陵有掩护射程；器械攻城强但平原怕骑兵冲。这让「在哪打、用什么兵」成为真决策。

### 14.2 天时 · 天气与季节系统
- **四季循环**：春/夏/秋/冬各带天气概率与**农时系数**（影响粮草产出），并触发季节事件（春汛涨水利水攻、严冬补给难）。
- **天气状态表**：

| 天气 | 对战场的影响（示例） |
|---|---|
| 晴 | 基准 |
| 雨 | 河流涨→**可水攻**；火攻失效；泥泞移速↓；骑兵↓；潮湿疫病↑ |
| 雪/严寒 | 行军损耗↑、补给耗↑；**干柴利火攻**；冻伤致士气↓ |
| 大风 | **火攻蔓延↑**；弓箭准确/射程↓；水攻势弱 |
| 大雾 | **视野↓（侦察失效）**；伏击/突袭/空城计加成，宜偷袭 |
| 酷暑/旱 | 火攻极效；体力↓；疫病↑；水源成必争 |

- **天气预报（庙算）**：谋士智力越高，可越准预测未来 N 时辰天气，让玩家「算准时机」火攻/水攻——呼应孙子「庙算」「用间」。

### 14.3 水攻 · 借洪水歼敌（你的核心诉求之一）
- **机制**：控制上游堤坝/水道/可决口点，对下游**低洼/盆地**敌区发动「水攻」：该区域驻军**持续伤害 + 士气崩 + 行军受阻（水障）**。
- **触发**：需上游控制权 +（雨季/已降雨水势更大更佳）；亦可人工决堤（消耗工兵/时间）。
- **双刃风险**：若己方单位也在低洼下游，同样受害——逼你算清地势。
- **史诗还原**：关羽水淹七军、曹操决泗水灌城。给用户「借天时地利发洪水」的极致爽点。
- **与天气联动**：雨/融雪时水攻伤害翻倍；旱季基本无效。

### 14.4 天险 · 难攻之地
- 山关、江防、长城、栈道、渡口享**天险惩罚**：攻方攻城值/行军速度被大幅削减（示例 ×0.5），守方享防御加成。
- **设计作用**：制造「一夫当关」的据点价值，把「堆兵力硬刚」逼成「迂回 / 暗度陈仓 / 水攻 / 断粮」等解法——丰富度由此而来。

### 14.5 人和 · 士气 · 忠诚 · 补给 · 同心
> 「人和」做成一套实时状态变量，与天时地利并重：

- **士气**：受 战况胜负、补给充足度、将领魅力/技能、天气（酷暑严寒↓）、地利（死地则战↑、散地则走↓，呼应孙子九地）影响；低士气→逃兵、战力↓
- **忠诚/民心**：占城后由 政治 属性与《司马法·军礼》科技决定；过低→叛乱/倒戈（呼应反间计、苦肉计）
- **同心/同袍**：同阵营、同乡、结义武将同队享光环加成；君主魅力提供全军光环
- **补给（粮道）**：断粮→士气崩、伤兵不恢复（「釜底抽薪」的线和维度）
- **三才评分**：出兵前「庙算」面板给出 **天时 / 地利 / 人和** 三维评分，直观兑现你要的「天时地利人和」。

### 14.6 环境结算器（实现核心）
战斗/行军前，由**环境结算器**综合：

```
最终修正 = 地形修正 × 天气修正 × 兵种-地形适配 × 天险惩罚 × 人和(士气)系数
```

- **全数据驱动**：`terrain.csv` / `weather.csv` / `season.csv` / `formations.csv` / `generals.csv` 配置，结算器读表计算，逻辑与表现分离，solo 易调易扩。
- **配表热加载（已实现）**：改 `data/*.csv` 后，Demo 点底部「配表」按钮即调用 `R.loadConfigFromServer('data/')` 重新拉取并热替换内存数据表（`reload()` 会就地回写 `TERRAIN/WEATHER/SEASON/FORMATIONS/GENERALS` 并刷新 `RULES` 引用），**数值即时生效、无需刷新页面**；Node 侧另有 `R.loadCSVFromDir(dir)` 便于写平衡脚本。调平衡时直接改 CSV 即可。
- 给玩家「庙算预览」：出兵前显示「此战 天时× 地利× 人和×」，让策略**可见**而非盲打。

### 14.7 给个人开发者的提醒
- **MVP 先做**：平原/山地/森林/河流 4 种地形 + 晴/雨/雾 3 种天气 + 士气系统，验证「地利人和」手感；沼泽损兵、水攻、四季再迭代加。
- **表现低成本高反馈**：地形用底色/图标区分，天气用全屏滤镜+粒子（雨丝/雪花/雾蒙），玩家一眼感知「天时变了」。

## 17. 六维优化（Phase B 已落地）

> 在「能跑」之后，对**手感 / AI 智商 / 外交深度 / 表现层 / 数值工具 / 架构**六个维度做了系统性打磨。所有增强仅扩展纯逻辑层 API，Demo 侧消费，无破坏式改动。

### 17.1 手感（策略可见、爽感清晰）
- **战斗节奏**：回合上限 `COMBAT_CFG.maxRound=10`；第 7 回合起进入猝死阶段，伤害递增（`suddenDeathRamp=0.3`），避免拖沓、保证收束。
- **伏兵未命中惩罚**：伏兵意图被识破（`ambushFailed`）时，攻方首回合士气重挫（`ambushMissMorale=-15`）且守军抢得先机——让「诡道」有真实风险与收益。
- **水攻持续回合**：依天气洪水系数决定持续时长（`floodDuration`），每回合 `tickFlood` 结算并递减，低洼敌区持续伤亡 + 士气崩。
- **士气传染**：`squadMoraleContagion` 让同袍情绪向均值靠拢；出现溃逃（<25）则连累全军（「兵败如山倒」），强化连锁崩溃的戏剧感。

### 17.2 AI 智商（单机不无聊的关键）
- **多线铺路**：每个行动源格独立记账（`usedSrc`），单 tick 最多 3 次行动（侵略/霸权），持续向中立/玩家扩张（实测 10 回合 116→209 格）。
- **包抄**：目标为敌对且相邻另有敌对 AI 时，占领评分 +4，形成夹击态势。
- **针对克制选阵法**：`chooseFormation` 按攻防/包抄/天险打分，克制守方风格 +40，对弓重包抄 +30，高阶阵（八阵）需智力解锁。
- **被夹击时撤退/求援**：领地骤减（`curLand < lastLand-6`）时，自动将某同盟设为最强 AI 的敌对以寻求外援。

### 17.3 外交深度（谋略感）
- **借道行军**：`validTarget` 允许借相邻同盟格迂回抵达目标，制造「假道伐虢」的腾挪空间。
- **附庸**：`setVassal` 单向臣服，双方均不可互攻（附庸体系）。
- **剧本事件**：`rivalAll`（众矢之的）、`allyAll`（结成讨贼同盟）等脚本化事件驱动名场面。
- **信誉影响议和**：`peaceAcceptChance` 受信誉调制（0.05~0.9），信誉越高越易议和，让「背信」有长期代价。

### 17.4 表现层（从色块 Demo 到可看）
- **地形图标化**：格子中央绘制地形汉字（山/林/水/城/泽…），缩放低于阈值自动隐藏。
- **天气全屏滤镜 + 粒子**：雨丝/雪点/雾团/风线/旱纹；晴无粒子、不开启 RAF 省电；约 70 粒子性能可控。
- **镜头缓动**：`panTo` + 缓动（lerp 0.18），战斗/事件自动运镜到焦点。
- **战斗动画与反馈**：统一 `animLoop`（相机缓动 + fx + 天气）驱动；`_flash`/文字飘字、屏幕抖动（`shake`）、暗角（`vignette`）提升打击感。

### 17.5 数值工具（调平不再靠手感）
- **自动平衡脚本** `tests/balance.js`：用 `loadCSVFromDir` 热加载 `data/*.csv`，跑大规模武将对战（全序对局）+ 地形防御优势（镜像对局）模拟，可复现（固定种子 mulberry32）。
- 输出：武将综合力排名与「过强/过弱」标记、地形守方胜率与「过防御/欠防御」标记、天气影响扫描；并生成 `tests/balance_report.md`。
- 改完 CSV 重跑即可对比均值与标准差是否收敛——数值调整有数据支撑。

### 17.6 架构与引擎移植接口
- **模块化拆分**：`src/` 下 UMD 模块（浏览器 `<script>` + Node `require` 双兼容），`rules.js` 为聚合入口：
  - `util.js`（clamp/pick）、`data.js`（配表 + 热加载 + 可变 `state`）、`combat.js`（环境/兵书/战斗/阵法）、`diplomacy.js`（关系模型）、`scenario.js`（剧本数据）。
  - 数据表经**可变 `state` 对象 + getter 暴露**，热加载（`reload`）即时生效，无需重建。
- **单元测试** `tests/unit.js`：隔离单测各子模块 + 聚合入口集成（44 项 PASS，失败非零退出，可接 CI）。
- **引擎移植接口（Godot / Cocos / Unity）**：外部引擎只需
  1. `require/import` 聚合入口拿到 `RULES`；
  2. 调用纯函数（`simulateCombat` / `makeUnit` / `createDiplomacy` / `chooseFormation` …）；
  3. 用 `loadCSVFromDir` / `loadConfigFromServer` 注入 `data/*.csv`；
  4. 渲染与输入由引擎自管——逻辑层**零 DOM 依赖**，可整体平移。
- **保留接口**：Godot `GDScript` 可经 `JavaScript Bridge` 调用本 `RULES`；Cocos `ts` 直接 `import`；后续可把 `simulateCombat` 等导出为 WASM/插件形态。
